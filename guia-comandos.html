O que é o Bash?

Bash é a linguagem do terminal — o interpretador que lê os comandos que você digita e os executa no sistema operacional.

Quando você abre o Terminal (Mac/Linux) ou o PowerShell (Windows), o que está rodando por baixo é o Bash. Ele é como um idioma universal para conversar com o computador sem usar interface gráfica.

Todo comando bash é basicamente: verbo + objeto + opções. Ex: cat (ler) + arquivo.txt (o quê).

💡 Analogia: Se o computador é uma cidade, o Bash é o rádio que você usa pra dar ordens diretas — sem precisar dirigir até cada lugar manualmente.

Cloudflare — Wrangler CLI

wrangler é a CLI oficial da Cloudflare. Instale com: npm install -g wrangler
Use no terminal para gerenciar Workers, D1, Pages e KV diretamente.
Comando O que faz
wrangler login Autentica sua conta Cloudflare no terminal
wrangler whoami Confirma qual conta está logada
wrangler deploy Publica o Worker atual na Cloudflare
wrangler dev Roda o Worker localmente para testar antes de publicar
wrangler tail Exibe logs em tempo real do Worker em produção
wrangler d1 create nome Cria um banco de dados D1
wrangler d1 execute nome --file=schema.sql Executa um arquivo SQL no banco D1
wrangler d1 execute nome --command="SELECT * FROM leads" Roda uma query SQL direta no D1
wrangler pages deploy ./dist Publica uma pasta estática no Cloudflare Pages
wrangler kv:namespace create "nome" Cria um namespace KV para armazenar chave-valor
wrangler secret put NOME Adiciona uma variável secreta ao Worker (ex: API keys)

Linguagem Natural — Templates de Prompt

Use estes modelos com Claude Code ou Claude + MCP Cloudflare.
A chave é: contexto + arquivo de referência + objetivo específico + restrições.
Intenção Template de Prompt
Criar banco D1 "Com base nos campos do formulário em orcamento.html, crie um banco D1 chamado leads-db com a tabela correta e me dê o SQL de criação."
Criar Worker "Crie um Cloudflare Worker que recebe um POST com os campos do formulário de contato.html, valida os dados e salva no D1 leads-db. Use CORS para aceitar requisições do domínio farpa.ai."
Ver leads "Execute um SELECT na tabela leads do banco leads-db e me mostre os últimos 10 registros em formato de tabela."
Debug Worker "O Worker api-leads está retornando erro 500. Veja os logs com wrangler tail e identifique a causa. O código está em worker.js."
Deploy Pages "Publique os arquivos da raiz do repositório no Cloudflare Pages. O projeto se chama farpa-ai e o branch principal é main."
Adicionar secret "Adicione a variável de ambiente WEBHOOK_URL como secret no Worker api-leads sem expor o valor no código."
Estrutura completa "A página simulador.html precisa salvar os resultados do usuário. Crie toda a infraestrutura: tabela D1, Worker com endpoint POST, e integre o fetch no JavaScript da página."

Anatomia de um Bom Prompt Técnico

Todo prompt eficiente tem 4 elementos:

1. Arquivo de referência → "Com base em orcamento.html..."
2. Ferramenta/serviço → "...crie um Worker na Cloudflare..."
3. Objetivo específico → "...que salva os dados no D1 leads-db..."
4. Restrição ou contexto → "...sem alterar o CSS existente / usando CORS / em português."

💡 Quanto mais específico o contexto, menos o Claude precisa adivinhar — e menos tokens você gasta.
